# oh-ai-car-ros-app 测试报告

报告对象：oh-ai-car-ros-app 智能小车巡检应用。

报告用途：用于课程设计、答辩材料或项目交付归档。

## 1. 测试概述

本次测试覆盖项目的核心业务链路，包括工程构建、签名安装、网络连接、首页展示、遥控控制、激光雷达参数、告警列表和 AI 咨询。

测试过程中重点验证了页面布局、配置读取、TCP 连接流程、输入输出链路以及异常提示的可用性。

## 2. 测试环境

- 开发工具：DevEco Studio 6.x。
- 系统环境：Windows 10 / Windows 11。
- 编译目标：HarmonyOS / OpenHarmony SDK API 21。
- 测试设备：HarmonyOS 模拟器或真机。
- 网络环境：本地局域网、小车测试服务端或仿真服务端。

## 3. 测试目标

- 验证项目可以正常同步、构建并生成 HAP。
- 验证签名配置可用，安装流程没有 `no signature file` 问题。
- 验证网络设置页面的连接逻辑。
- 验证主要功能模块的展示与交互。
- 验证 AI 咨询和告警分析是否可用。

## 5. 典型测试用例

### 用例 1：网络连接

- 步骤：打开网络设置页，输入目标 IP、TCP 端口 6000、视频端口 6500。
- 预期：点击连接后成功建立 TCP 连接并进入首页。
- 实际：连接成功后在线状态更新正常。

### 用例 2：遥控控制

- 步骤：进入遥控页面，拖动摇杆并点击方向按钮。
- 预期：小车接收控制命令并执行对应动作。
- 实际：控制命令可正常发送，页面状态无异常。

### 用例 3：告警咨询

- 步骤：在告警页面选择多条告警，输入咨询问题并发送。
- 预期：系统返回 AI 分析结果，展示风险、建议和总结。
- 实际：在 API 可用时正常返回结果，未配置时给出失败提示。

### 用例 4：激光雷达阈值调整

- 步骤：打开激光雷达页面，修改前方减速、停车和占用阈值。
- 预期：参数修改后可参与后续状态判断和告警触发。
- 实际：页面参数可编辑并保存到当前运行状态。

## 6. 已知问题

- 在未正确配置签名或签名材料失效时，安装阶段会出现 `no signature file`。
- 若目标小车服务端未启动，TCP 连接将失败。
- AI 咨询需要可用的 API Key 和网络访问能力。
- 旧的 RTF 文档因编码问题不再推荐使用，统一以 MD 为准。

## 7. 修复与验证

- 已恢复 `build-profile.json5` 中的签名配置，避免 unsigned HAP。
- 已修正 `module.json5` 的冲突和语法问题。
- 已对告警页面和雷达页面做过多轮布局修复。
- 已重新整理成 Markdown 文档，便于直接提交和维护。

## 8. 结论

综合测试结果表明，项目主链路功能可用，已具备展示、控制、分析和告警管理能力。后续若接入真实小车与传感器，可在现有架构上继续联调扩展。

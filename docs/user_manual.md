# oh-ai-car-ros-app 项目使用手册

适用对象：项目开发、联调、测试和演示人员。

项目类型：HarmonyOS / ArkTS / ArkUI 智能小车巡检与控制应用。

## 1. 项目概述

本项目用于仓库或通道巡检场景下的小车远程控制、环境监测、告警分析与 AI 咨询辅助。应用以网络连接为入口，联动视频预览、遥控操作、雷达检测、视觉识别和告警管理等功能模块。

在当前版本中，应用已经具备基础巡检展示、异常信息汇总、AI 分析、网络连接与远程控制能力，后续可继续扩展与 Jetson 小车、雷达设备及视频服务的联调。

## 2. 功能组成

- 首页：展示巡检状态、在线状态、关键模块入口和告警摘要。
- 网络设置：配置小车 IP、TCP 端口、视频端口并建立连接。
- 遥控页面：支持摇杆控制、方向控制、视频预览和手动接管。
- 环境监测：展示温湿度、烟雾等环境数据及异常状态。
- 激光雷达页面：支持前方通道占用、左右货架距离、避障阈值与告警策略。
- 视觉识别页面：用于人员、障碍物和画面异常识别结果展示。
- 告警页面：支持告警列表、引用咨询、AI 汇总分析和模板问答。
- 系统监控：用于展示小车运行状态、巡检进度和任务执行情况。

## 3. 运行环境要求

- DevEco Studio 6.x 或与项目兼容的版本。
- HarmonyOS / OpenHarmony SDK API 21。
- Windows 10 / Windows 11 开发环境。
- 已安装 DevEco Studio 自带 Node.js，或在 `local.properties` 中正确配置 `nodejs.dir`。
- 可用的 HarmonyOS 手机、模拟器或测试设备。
- 如需联机调试，目标小车端服务必须已启动，并允许当前设备访问其 IP 与端口。

## 4. 目录说明

- `AppScope`：应用级资源和全局配置。
- `entry`：主应用模块，包含页面、组件、TCP 通信、工具类和资源。
- `Rocker`：摇杆组件模块。
- `docs`：生成的项目文档。
- `build-profile.json5`：项目级构建与签名配置。

## 5. 本地配置

项目根目录需要配置 `local.properties`，保证 DevEco Studio 能定位 SDK 和 Node.js。示例如下：

```properties
sdk.dir=D:/OpenHarmony/SDK
nodejs.dir=D:/Program Files/DevEco Studio/tools/node
```

建议使用正斜杦，避免 Windows 路径转义问题。若本地路径与示例不同，以你机器上的实际安装位置为准。

网络连接相关配置通常保存在应用内偏好设置中，默认值如下：TCP 端口 6000，视频端口 6500，IP 需按目标小车实际地址填写。

## 6. 构建与运行流程

1. 使用 DevEco Studio 打开项目根目录。
2. 首次打开后同步工程，等待依赖下载完成。
3. 确认 `build-profile.json5` 中签名配置已恢复为可用状态。
4. 执行 `Build -> Clean Project`，清理旧构建产物。
5. 执行 `Build -> Build Hap(s)/APP(s) -> Build Hap(s)`，生成安装包。
6. 连接模拟器或真机后，点击 `Run` 运行应用。

如果出现编译失败，优先查看 hvigor 输出、签名配置和模块 JSON5 配置是否存在冲突标记或字段缺失。

## 7. 网络连接使用说明

- 打开应用后会先进入网络连接页。
- 填写小车 IP 地址。
- 填写 TCP 端口，默认使用 6000。
- 填写视频端口，默认使用 6500。
- 点击连接后，应用会尝试建立 TCP 连接并初始化视频组件。
- 连接成功后可进入首页和遥控页面。
- 如果连接失败，先检查目标设备地址是否能访问，再检查小车服务端是否已启动。

注意：6000 是当前小车控制协议对应的端口号，不代表其他端口不能使用，而是本项目的默认控制通道。

## 8. 首页说明

- 首页展示当前在线状态、巡检状态、任务进度、异常告警摘要和模块入口。
- 模块入口包括环境感知、SLAM 导航、激光雷达、视觉识别和远程接管。
- 页面上的连接状态适合用于快速判断小车当前是否在线。
- 首页底部的告警预览用于提示最近的重点异常。

## 9. 遥控页面说明

- 遥控页面支持摇杆控制和方向按钮控制。
- 摇杆用于方向控制，按钮用于前进、后退、转向、刹车等快捷动作。
- 视频预览区域用于观察小车前方实时画面。
- 高级调节区可以对轮速或控制参数做进一步调试。
- 执行控制前，请确保小车已经在线并且 TCP 连接建立成功。

## 10. 激光雷达页面说明

- 页面用于展示前方通道、左右货架距离和通道状态。
- 支持设置前方减速阈值、停车阈值和通道占用判定阈值。
- 支持短时障碍、长时障碍和通道堵塞分类。
- 支持后续与仓库通道安全检测场景扩展对接。

## 11. 告警页面说明

- 告警列表按时间和级别展示巡检异常记录。
- 支持选择多条告警进行引用咨询。
- 支持 AI 汇总分析，输出风险级别、处置优先级和建议。
- 支持模板问答，提高常见问题咨询效率。
- 若 DeepSeek API Key 未配置，AI 咨询会返回失败提示。

## 12. 常见问题与处理

- 安装失败 `no signature file`：检查 `build-profile.json5` 中 `signingConfigs` 是否恢复，确保 `products.default` 仍然引用 `default` 签名。
- 连接失败：检查 IP、TCP 端口、防火墙以及小车服务端是否真正启动。
- 视频黑屏：确认视频端口、视频服务地址以及当前网络是否可达。
- AI 咨询失败：确认 `config.env` 中的 DeepSeek API Key 已配置且网络可访问。
- 页面显示异常：优先检查是否存在旧缓存、冲突标记或资源文件未同步。

## 13. 使用建议

- 首次运行建议先完成网络配置，再进行视频与遥控联调。
- 签名、SDK 路径和 Node 路径属于本地环境配置，不建议提交到公共仓库。
- 涉及联机设备调试时，先用模拟器验证 UI，再切换到真实设备。
- 每次调整 TCP 协议或资源文件后，建议重新 `Clean` 和 `Sync` 一次。
